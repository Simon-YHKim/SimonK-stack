# 내부 / 본인 타임라인 메모

대응 중 실시간으로 적는 작업 로그. 솔로 빌더는 이게 곧 postmortem 원재료다.
`scripts/incident-log.sh` 가 이 형식으로 누적한다 — 수동 작성 시 동일 형식 유지.

---

- **제목**: {한 줄 증상}
- **SEV**: {1/2/3/4}
- **T0 (감지)**: {YYYY-MM-DD HH:MM KST}
- **대응자**: {이름}
- **상태**: 조사중 / 완화중 / 모니터링 / 해결

## 타임라인 (위에서 아래로 추가)

| 시각(KST) | 무엇을 했나 | 결과/관찰 |
|---|---|---|
| HH:MM | 감지 — {알림/사용자 제보/직접 발견} | |
| HH:MM | `git log` 최근 변경 확인 | 마지막 배포 {커밋/시각} |
| HH:MM | SEV {n} 판정 | 근거: {영향×범위} |
| HH:MM | 완화 액션: {FLAG KILL / ROLLBACK / DEGRADE / HOTFIX} | 효과: {영향 사라짐? 신호?} |
| HH:MM | 효과 확인 — {재현/로그/에러율} | |
| HH:MM | 모니터링 시작 (15분) | |
| HH:MM | 종료 — 재발 없음 | |
