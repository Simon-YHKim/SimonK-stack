# DSAR 응답 체크리스트 — 정보주체 요청 처리

> 정보주체(이용자)가 자기 데이터에 대해 열람·정정·삭제·이동·처리정지를 요청했을 때의 대응 절차.
> 데이터 위치는 `data-flow.md` §4 DSAR 매핑에서 가져온다.

## 요청 유형별 처리

| 요청 유형 | 한국(PIPA) | GDPR | 응답 기한 |
|---|---|---|---|
| 열람 (Access) | §35 | Art.15 | 10일 / 30일 |
| 정정·삭제 (Rectify/Erase) | §36 | Art.16-17 | 10일 / 30일 |
| 처리정지 (Restrict) | §37 | Art.18 | 10일 / 30일 |
| 이동 (Portability) | (제한적) | Art.20 | 30일 |

## 처리 절차

- [ ] 1. 요청자 신원 확인 (계정 본인 확인 — 과잉 수집 금지)
- [ ] 2. 요청 유형 분류 + 접수 시각 감사로그 기록
- [ ] 3. `data-flow.md` 위치 맵으로 대상 데이터 전수 식별
  - [ ] 자체 DB 테이블/컬럼
  - [ ] 외부 처리자(3rd party) — 각 처리자 삭제/내보내기 API 호출
  - [ ] 백업·로그·캐시
- [ ] 4. 법적 보존 의무 데이터는 제외 (전자상거래법 등) — 사유 고지
- [ ] 5. 실행 (열람: export / 삭제: hard or soft delete) + 감사로그 기록
- [ ] 6. 처리자에게 전파 확인 (삭제가 다운스트림까지 반영됐는지)
- [ ] 7. 요청자에게 결과 통지 + 처리 내역 보관

## 주의

- 삭제는 fail-closed: 한 처리자라도 삭제 실패 시 "완료" 통지 금지.
- 모든 단계는 감사로그 필수 — 사후 규제 대응 증빙.
- 아동/대리인 요청은 보호자 권한 확인 절차 추가.
